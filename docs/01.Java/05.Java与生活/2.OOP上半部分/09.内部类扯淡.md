# 内部类扯淡

！：在实际开发过程中，内部类并不常用，因为我们总是会把一个类单独写一个文件。

> 一个文件里只能有一个公共类?

```Java
public class Sun {
    // 内部类
    class Earth {
        // ...
    }
}
```

不推荐使用"内部类"这种东西，这样写的情况少之又少。

内部类的可维护性并不好。

> 而"匿名内部类"会常用一些。

用案例来讲"内部类"会比较复杂，把书上的原理看懂就行。。。
